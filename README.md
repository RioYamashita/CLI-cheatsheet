# CLI-cheatsheet

CLIコマンドの学習用ノートをまとめるリポジトリ。  
あわせて、GitHubの主要機能  
（Code / Issues / Pull Requests / Actions）を実践的に学ぶことを目的とする。

---

## 目的
- GitHubの主要機能（Code / Issues / PR / Actions）の使い方を理解する
- CLIコマンドを整理し、言語化・ドキュメント化する練習を行う
- Issue駆動で学習し、履歴として残す習慣を身につける

---

## 構成
- `bash/`：bashの基本コマンド
- `git/`：git関連コマンド
- `docker/`：docker / container系CLI
- `sample.txt`：練習用ファイル

---

## 学習・運用ルール

### Issues（学習テーマの管理）
- 学習したい内容は **1テーマ = 1 Issue** として作成する
- Issueには「概要 / やること / 完了条件」を記載する
- 学習が完了したら、対応するPRで close する

### Pull Requests（学習ログの記録）
- Issue対応の作業は必ずブランチを切って行う
- 変更内容・学んだことをPRの説明欄にまとめる
- 関連するIssue番号をPRに紐づける（例：`close #1`）

### Code（ノートの管理）
- CLIコマンドはMarkdownで整理する
- コマンドの目的・使い方・例をセットで記載する
- ディレクトリ構成から学習内容が分かるようにする

### Actions（自動チェック）
- PR作成時に簡単なチェックを自動実行する
- READMEや学習ノートの存在確認など、最小構成から始める
- CI/CDの概念理解を目的とし、過度な自動化は行わない

## このリポジトリで身につけたいこと
- GitHubを使った学習フローの理解
- Issue → PR → merge の一連の流れ
- 変更履歴を意識した学習・記録の習慣化
